**Важное**:
1. Финальный проект $-$ это "экзамен" по этому курсу, то есть последний элемент контроля. 
2. Вес проекта $-$ 3 балла от итоговой оценки за курс. 
3. Дефолтно проект считается индивидуальным заданием[^1]. 
4. Вы можете выбрать один из шаблонов из таблички / списка внизу, а можете предложить свою тему. Один шаблон могут выбрать несколько человек, но варьирующиеся элементы шаблона должны быть у каждого свои (если не указано, что тема для одного человека). Предлагая свою тему проекта, вы должны понимать, что проект должен приблизительно вписываться в одну из схем: сайт, исследовательский проект, API или tg-бот.

### *Ориентировочное* соответствие задач баллам:

#### **Сайт**:
- _2 балла_<br>Cайт работает, имеет понятный для пользователя интерфейс: описание (в том числе, в `README` в папке с проектом), меню, переходы по ссылкам и т.д.
- _2 балла_<br>Используется морфологический и_ли синтаксический парсинг (и_ли какие-то из NLP-инструментов), построены графики, есть описание графиков или любых других выводимых данных.
- _2 балла_<br>Данные скачиваются и_ли сохраняются в базу данных.
- _2 балла_<br>Код проекта выложен в репозиторий, а также на `pythonanywhere` или на любой другой хостинг (или ваш собственный сервер), т.е. сайт должен работать не только локально (ссылка должна быть в `README` в папке с проектом).
- _2 балла_<br>Что-то сложное и оригинальное: элементы машинного обучения, сложные графики, статистика и т.д.

#### **Исследовательский проект**:
- _2 балла_<br>Код работает, цели и задачи чётко сформулированы, продумана последовательность действий, каждый шаг прокомментирован.
- _2 балла_<br>Есть краулер и_ли сохранение в базу данных.
- _2 балла_<br>Есть морфологический и_ли синтаксический парсинг (и_ли используются какие-то из NLP-инструментов) и_ли натренирована модель.
- _2 балла_<br>Есть визуализация результатов с описанием.
- _2 балла_<br>Что-то сложное и оригинальное: элементы машинного обучения, сложные графики, статистика и т.д.

#### **API**:
- _2 балла_<br>API работает, имеет понятный для пользователя функционал: описание (в том числе, в `README` в папке с проектом и на автоматической странице с документацией), вывод списка команд, валидация данных и т.д.
- _2 балла_<br>Есть краулер и_ли сохранение в базу данных.
- _2 балла_<br>Есть морфологический и_ли синтаксический парсинг (и_ли используются какие-то из NLP-инструментов) и_ли натренирована модель.
- _2 балла_<br>Код проекта выложен в репозиторий, а также на `pythonanywhere` или на любой другой хостинг (или ваш собственный сервер), т.е. API работает не только при локальном запуске (ссылка должна быть в `README` в папке с проектом).
- _2 балла_<br>Что-то сложное и оригинальное: элементы машинного обучения, сложная статистика и т.д.

#### **Телеграм-бот**[^2]:
- *3 балла*<br>Бот работает, имеет понятный для пользователя интерфейс: описание (в том числе, в `README` в папке с проектом), команды, помощь и т.д.
- *3 балла*<br>Есть морфологический и_ли синтаксический парсинг (и_ли используются какие-то из NLP-инструментов) **и** натренирована модель.
- *2 балл*<br>Есть интересный функционал: аудио, картинки, кнопки.
- *2 балла*<br>Код проекта выложен в репозиторий, а также на `pythonanywhere` или на любой другой хостинг (или ваш собственный сервер), т.е. бот должен работать не только при локальном запуске (ссылка должна быть в `README` в папке с проектом).

Еще возможные темы:

1) Граф знакомств из ВК или граф общих пользователей разных групп, какой-то анализ социальных сетей (графов).
2) Контрастивное сравнение 2-х ютуб-каналов / сообществ ВК. 
3) Исследование Евровидения: сравнение текстов; построение модели, объясняющей, кто какие кому оценки ставит
4) Исследование характерных региональных слов в ВК.
5) Анализ тональности комментариев ютуб-каналов / сообществ ВК.
6) "Болталка"-бот, обученный на большом корпусе литературных текстов, чтобы отвечать как-то подходяще.

8) Анализ тональности новостей, например, динамика во времени, сравнение источников, контекст использования определенных имен (например, взять разметку через NER).
